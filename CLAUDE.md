# CLAUDE.md — 더서브온 AI 자가진단 대시보드 프로젝트 지침

Claude Code가 이 저장소에서 세션을 시작하면 이 파일을 자동으로 읽습니다. 아래 순서를 반드시 지키십시오.

## 0. 시작 전에 반드시 할 일

1. `PRD.md` 전체를 끝까지 읽으십시오. 이 프로젝트의 유일한 요구사항 원천입니다.
2. Plan 모드로 구현 계획을 세우고, `TodoWrite`로 `PRD.md` 12장의 마일스톤(M1~M9)을 태스크로 등록하십시오.
3. `demo-assets/diagnosis_sample.json`과 `demo-assets/사용가이드.md`를 확인하십시오. 실제 더서브온 사전진단 실측 데이터와 그 진단 결과 예시가 들어 있습니다. 프로젝트 스캐폴딩 후 `public/diagnosis_sample.json`으로 복사해 F12(샘플 진단 불러오기) 기능에 연결하십시오.

## 1. 이 프로젝트의 성격 (콘텐츠 팩토리 앱과 다른 점)

이 앱은 "만드는 도구"가 아니라 "진단하는 도구"입니다. 최우선 목표는 **더서브온 담당자가 컨설팅 종료 후에도 매달 스스로 반복 사용할 수 있는 완성도**이지, 특정 날짜의 1회성 데모가 아닙니다. 다만 F12(샘플 진단 불러오기)만큼은 데모/온보딩 겸용으로 반드시 구현하십시오.

## 2. 절대 잊으면 안 되는 것 (요약)

- Codex가 만드는 짝 프로젝트와 **기능 요구사항이 100% 동일**해야 합니다(`PRD.md` 최상단 참조). 임의로 기능을 추가/축소하지 마십시오.
- `PRD.md` 8장의 두 프롬프트(콘텐츠 감사 서브 프롬프트, 종합 진단 마스터 프롬프트)와 그 안의 **판정 기준 수치(70%, 50%, 10%p, 30일/90일 등)를 절대 임의로 바꾸지 마십시오.** 이는 컨설팅팀이 사람이 직접 진단하며 사용한 논리를 그대로 코드화한 것입니다.
- 네이버 검색 순위·SNS 인사이트의 완전 자동 크롤링은 **구현 금지**입니다(11장 참조). F5처럼 "검색 URL 자동 생성 + 사람이 확인해 입력"하는 반자동 패턴만 사용하십시오.
- API 키(`ANTHROPIC_API_KEY`)는 서버 사이드에서만 사용하고 절대 클라이언트에 노출하지 마십시오.
- D1·D5처럼 사람이 입력해야 하는 값은, 입력하지 않았을 때 "미입력"으로 명확히 구분 표시해야 합니다(빈 값을 실측한 것처럼 보여주면 안 됨).

## 3. 기술 스택 (고정, 임의 변경 금지)

Next.js 14+ (App Router) + TypeScript + Tailwind CSS + `@anthropic-ai/sdk` + `fast-xml-parser`(RSS 파싱) + `recharts`(차트) + 브라우저 `localStorage`. 자세한 내용은 `PRD.md` 11장 참조.

## 4. 환경변수

```
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-sonnet-5
```
`.env.example`을 만들고, `.env.local`은 `.gitignore`에 포함하십시오.

## 5. 진행 순서

`PRD.md` 12장 "개발 절차"의 M1~M9를 순서대로 따르십시오. 각 마일스톤 종료 시 `npm run dev`로 직접 동작을 확인하십시오.

## 6. 완료 기준

`PRD.md` 14장 "완료 기준(Definition of Done)" 체크리스트를 전부 통과해야 작업이 끝난 것입니다.
